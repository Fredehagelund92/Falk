# Security Policy

## Supported Versions

Security updates are provided for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in falk, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

### How to Report

1. **GitHub Security Advisories** (preferred): Go to the [Security](https://github.com/Fredehagelund92/Falk/security) tab and click "Report a vulnerability".
2. **Email**: Send details to [frederik.hagelund@gmail.com](mailto:frederik.hagelund@gmail.com) with a descriptive subject line.

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Status updates**: As needed until resolved

### Acknowledgment

We may acknowledge reporters in release notes or a dedicated security acknowledgments page, unless you prefer to remain anonymous.
